# Skills 激励体系设计

> 讨论日期：2026-03-19

---

## 一、为什么不采用一口价买卖

一口价适合「买完即用、价值明确」的商品，但 Skills 有三个特殊性，使其不适合这种模式：

1. **价值是动态的**：同一个 Skill，重度用户和轻度用户感知到的价值差异可能高达 10 倍
2. **质量需要使用才能验证**：用户购买前无法准确判断 Skill 是否适合自己的场景
3. **Skills 会被作者持续迭代**：一口价的定价逻辑与持续更新的软件工具不匹配

因此，推荐方向是**不让用户直接对单个 Skill 付费**，而是设计平台层面的统一分发与收益机制。

---

## 二、激励体系设计（三层结构）

### 第一层：平台会员制 + 创作者收益池

**用户侧**

- 不对单个 Skill 单独付费，统一通过「点点 AI Pro 会员」解锁所有 Skills 的无限用量
- 免费用户享有次数限制（如每个 Skill 每月 5 次体验额度）

**创作者侧**

- 平台将会员收入的固定比例注入「创作者收益池」，按月分配
- 分配权重综合以下指标：

| 指标 | 说明 |
|------|------|
| 安装量（存量） | Skill 被安装的累计用户数 |
| 月活跃使用次数 | 真实使用行为，防止刷量 |
| 用户留存率 | 安装后持续使用的比例 |
| Skill 评分 | 用户主动评分与评论质量 |

**核心优势**：创作者无需自行定价、处理客服或退款，专注把 Skill 做好即可，平台负责货币化。逻辑上类似 Spotify 对独立音乐人的收益分配模式。

---

### 第二层：Skill 笔记的流量激励

小红书已有成熟的「内容带货 → 佣金」基础设施，Skills 可直接复用这套逻辑。

**创作者推广自己的 Skill（一方推广）**

- Skill 笔记带有平台特殊标记，获得额外的自然流量加权（类比商品笔记扶持政策）
- 创作者后台可查看「每篇笔记带来的安装转化数」，形成直接反馈，激励持续发布内容

**第三方推广（三方佣金，解决非创作者的推广动机）**

- 任何用户均可为感兴趣的 Skill 生成专属「推广链接」
- 通过该链接安装并产生使用行为后，推广者获得相应佣金
- 佣金来源：从平台补贴或创作者收益中划拨一定比例
- 该模式与小红书现有「好物推荐」逻辑完全一致，用户已有认知基础，迁移成本极低

---

### 第三层：平台前期补贴策略

冷启动阶段建议**完全免费 + 用创作者激励代替用户付费**，分三个阶段推进：

| 阶段 | 时间参考 | 用户体验 | 创作者激励来源 |
|------|---------|---------|--------------|
| 冷启动期 | 0–6 个月 | 全部免费，无限制 | 平台直接发放现金或流量激励 |
| 成长期 | 6–18 个月 | 免费用户有次数上限 | 会员收入池 + 平台补贴混合 |
| 成熟期 | 18 个月以后 | Pro 会员制稳定运营 | 主要来自会员收入池 |

**冷启动期补贴的核心逻辑**：平台激励促使创作者愿意花精力做高质量 Skill，并主动发笔记推广。发内容不仅带来粉丝增长，还有现金回报。这个内容供给飞轮一旦启动，用户侧使用习惯随之建立，后续付费转化就有了基础。

---

## 三、待讨论：创作者分级问题

头部博主的 Skill 天然具备流量优势，在收益池分配机制中可能形成「强者恒强」的局面，导致长尾创作者失去积极性，整个市场被少数大 V 垄断。

可能的应对方向：
- 对新发布 Skill 设置「新人扶持期」，给予额外流量加权
- 在分配公式中引入「增长率」指标，奖励快速成长的 Skill，而非仅奖励绝对体量
- 按赛道分类，保证每个垂类都有上升空间，避免跨赛道的直接竞争挤压

## 四、与现有小红书体系的对照

| 电商概念 | Skills 对应概念 |
|---------|----------------|
| 商品 | Skill |
| 购买 | 安装 / 使用 |
| 商品笔记 | Skill 笔记 |
| 带货佣金 | Skill 推广佣金 |
| 店铺主页 | 创作者主页 |

Skills 激励体系的设计原则：**尽量复用小红书用户已经熟悉的内容-商业化行为路径，降低认知和行为门槛。**
