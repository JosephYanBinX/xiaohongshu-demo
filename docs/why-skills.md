# 为什么要用 Skills？——以「菜单助手」为例

## 两种方式的对比

| | ✅ 使用 Skills | ❌ 不使用 Skills |
|---|---|---|
| **触发方式** | 一句话：「输出这周的菜单」 | 打开小红书搜索，或打开 AI 手动描述需求 |
| **菜单内容** | 自动生成，已结合你的口味偏好、厨具条件、荤素搭配 | 自己从十几篇笔记里挑，手动安排到每一天 |
| **购物清单** | 自动按食材分类，标注用量，直接拿去超市 | 自己逐条整理，估算用量，容易漏买 |
| **制作教程** | 每道菜附上小红书高赞做法链接，点开即看 | 再次搜索，一道菜一道菜找教程 |
| **时间成本** | 30 秒以内 | 30–60 分钟 |
| **个性化程度** | 越用越懂你，打分后自动调整偏好 | 每次从零开始，没有记忆 |
| **可迭代性** | 不喜欢某道菜 → 打分 → 下次自动规避 | 下周重新来一遍，没有任何积累 |

## 核心差异

Skills 不是「帮你做一件事」，而是**把一套需要反复执行的任务流，封装成一个记得住你、持续进化的私人助手**。

普通 AI 问答的问题在于：它没有记忆，没有上下文，也不了解你的具体情况——你每次都要重新解释一遍自己的需求。Skills 解决的正是这个问题。
