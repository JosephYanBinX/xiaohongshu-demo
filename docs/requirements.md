## 一、产品定位

**点点 AI** 是小红书内置的 AI 助手，目标是从传统"问答聊天机器人"升级为**能真正帮用户完成任务的 AI**。完成任务的核心机制是 **Skills（技能）**。Skills 由社区用户自由创建和分享，任何人都可以是 Skill 作者。小红书上已有大量经验、攻略类内容，本质上就是"完成某任务的方法论"——Skills 让这些内容从**可阅读**升级为**可执行**。

---

## 二、核心概念

| 概念 | 说明 |
|------|------|
| **Skill** | 针对特定场景、可被 AI 执行的任务方法论。例：`菜单助手`（每周菜单规划） |
| **Skill 笔记** | 在小红书 Feed 中发布的 Skill 介绍帖，是 Skill 的发现和分发渠道 |
| **Skill 市场** | 在点点 AI 的 Skills Tab 中展示所有公开发布的 Skills |
| **已安装 Skills** | 用户添加过的 Skills，出现在消息 Tab 和点点 Skills Tab 顶部 |

---

## 三、页面导航与入口关系

```
小红书首页
├── 左上角气泡icon（唯一入口）→ 点点AI主页
│   ├── Tab：对话  →  与点点的主对话流
│   └── Tab：Skills
│       ├── 顶部：我的已安装Skills（横向图标行）→ 各Skill对话页
│       ├── 主体：Skill市场（列表，无限加载，支持搜索/筛选）→ Skill详情页
│       └── 底部大按钮：创建Skill → Skill创作流程
│
├── Feed中的Skill笔记（外观同普通笔记）
│   └── 点击 → Skill详情页
│       ├── 展示：基本信息、简介、使用说明、评论、评分
│       └── 右上角"添加"按钮 → 底部弹层"添加成功 | 去使用"
│           └── 点击"去使用" → 该Skill对话页
│
└── 底部导航：消息Tab
    └── 已安装Skills以独立会话出现在列表中 → 各Skill对话页
```

## 五、页面详细设计

### 5.1 小红书首页 Feed
- Skill 笔记在 Feed 中**外观与普通笔记完全相同**，不做特殊区分
- 参考：`00-home-feed.jpg`

---

### 5.2 笔记详情页（普通 vs 商品 vs Skill）

底部操作栏左下角区分三种类型：

| 类型 | 左下角 |
|------|--------|
| 普通笔记 | 无 |
| 商品笔记 | "前往[平台]购买"按钮（参考 `02-note-detail.jpg`） |
| Skill 笔记 | "查看 Skill 详情"按钮，点击 → 进入 **Skill 详情页** |

其余结构（图片、标题、正文、点赞/收藏/评论）完全相同。

---

### 5.3 Skill 详情页

**入口：** Skill 笔记详情页底部按钮 / Skill 市场卡片点击

**页面内容：**
- 顶部：返回 | "Skill 详情"标题 | 分享
- Skill icon（圆形，demo 用 emoji 代替）+ Skill 名称 + 作者
- 简介文字
- **使用说明**（即 Skill 说明书内容）：设定最大高度，超出折叠，展示"展开全部"
- 基本数据：安装人数、评分（星级）、创建时间
- **评论区**：用户对该 Skill 的评价（同笔记评论样式）
- 底部固定操作栏：
  - **未安装状态**：绿色大按钮"＋ 添加到我的 Skill"；点击后同时将该 Skill 加入消息 Tab 会话列表
  - **已安装状态**：左侧"取消安装"（描边按钮）+ 右侧"去使用"（绿色实心按钮）；点击"取消安装"将 Skill 从已安装列表移除；点击"去使用"进入该 Skill 对话页

---

### 5.4 Skill 对话页

**入口：**
- Skill 详情页 Toast 中"去使用"
- 消息 Tab 中对应 Skill 会话
- 点点 Skills Tab 顶部已安装 Skills 图标

**页面结构（主题色：绿色，参考 `01-diandian-chat.jpg`）：**
- 顶部：`×` 关闭 | Skill 名称（绿点状态）| `...` 更多（含"使用说明"入口，点击弹出底部浮层）
- 对话流顶部：**使用说明块**（卡片，默认折叠，展示摘要，可展开查看完整说明书内容）
- 主体：对话流
  - **消息动画**：进入页面后消息逐条出现，非全量渲染。AI 消息先显示打字指示器（三点动画约 1.3s），再替换为实际内容；用户消息从下方渐入；每条消息之间有间隔（AI 后 700ms，用户后 400ms）
- 底部：输入栏
- **使用说明浮层**：从底部滑入的半屏浮层，展示完整说明书内容，支持关闭

---

### 5.5 消息 Tab 页

**结构参考 `03-messages-tab.jpg`，新增 Skill 会话：**
- 快捷分类三格：赞和收藏 / 新增关注 / 评论和@
- 会话列表混排：
  - **点点 AI**（固定置顶，带 `ai` 角标）
  - **已安装的各 Skill**（带 `ai` 角标，以 Skill 名称为会话名，Skill icon 为头像）
  - 普通私信、群聊、系统消息

### 5.7 Skill 创作流程

**入口：** Skills Tab 底部"+ 创建 Skill"按钮

#### Step 1：基本信息
- 输入 Skill 名称（必填）
- 输入 Skill 简介（必填）
- 右上角"存草稿"按钮

#### Step 2：与 AI 对话定义 Skill
- AI 主动引导提问，交互式完善 Skill 内容
- 也支持上传 `SKILL.md` 文件（快捷方式）
- 右上角"存草稿"按钮
- **Demo 模拟方式**：进入页面时自动显示第一条用户消息；点击屏幕推进一轮（AI 回复前短暂显示"思考中..."动画约 1s）；依次点击推进直到出现试用卡片

**Demo 对话脚本：** 见 [`docs/content/creation-dialogue-meal-plan.md`](./content/creation-dialogue-meal-plan.md)

**对话结束后的交互：**
1. AI 最后一条消息下方出现**试用卡片**（圆角卡片，绿色主题）
   - 内容：Skill icon（🍽️）+ 名称 + 简介 + "点击试用"提示
   - 点击卡片 → 跳转到该 Skill 的对话页（试用模式）
2. 用户试用完毕，返回创作对话流
3. 对话流底部出现绿色**"下一步"**按钮
4. 点击"下一步" → 进入 Step 3

#### Step 3：Skill 说明书审阅
- AI 自动生成说明书（功能介绍、使用方法、命令格式、示例等）
- 说明书内容较长时，页面内可滚动阅读
- 支持手动编辑任意内容
- 底部"下一步"按钮

#### Step 4：最终确认 & 发布
- 可再次修改 Skill 名称、简介
- 展示 Skill 摘要信息（icon、名称、简介、命令格式）
- **Skill Icon**：AI 自动生成一张图标，demo 中用 emoji 占位，交互不实现
- "发送给朋友测试"按钮（demo 中可做成样式，不实现功能）
- "发布"按钮 → 发布成功
  - 该 Skill 出现在创作者的已安装 Skills 列表
  - 出现在 Skill 市场，所有人可搜索和安装
