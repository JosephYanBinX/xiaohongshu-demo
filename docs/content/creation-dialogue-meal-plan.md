# 模拟对话脚本：创建「菜单助手」Skill

> 用于 Skill 创作流程 Step 2 的 demo 模拟对话。
> 交互方式：进入页面时自动显示第一条用户消息，
> 之后每次点击屏幕推进一轮对话，直到出现试用卡片。

---

## 对话轮次

### 消息 1（自动显示，用户发出）
```
我想做一个帮我规划每周菜单的 skill，我家有个阿姨负责做饭，
每周需要提前规划周一、二、四、五的菜，每天三道菜。
```

---

### 消息 2（点击后，AI 回复）
```
好的！我来帮你搭建这个菜单规划 Skill 🍽️
先问你几个关键问题：

1. 菜品偏好有什么要求吗？比如口味、忌口、擅长的菜系？
2. 食材从哪里采购？需要考虑购物清单吗？
3. 阿姨的厨艺水平大概怎样，有没有做不了的烹饪方式？
```

---

### 消息 3（点击后，用户回复）
```
家里有炒锅、空气炸锅、高压锅，没有烤箱。
口味没有特别限制，但希望荤素搭配。
需要生成购物清单，食材去超市买。
```

---

### 消息 4（点击后，AI 回复）
```
明白了。关于菜谱来源和更新机制——

这个 Skill 是每次使用时从小红书搜菜谱，
还是维护一个本地菜谱库、定期更新？

后者体验更好，因为可以记住你喜欢/不喜欢哪些菜 😊
另外，我可以帮你在小红书创建一个「本周菜单」收藏专辑——
平时刷到觉得好吃的菜，直接收藏进这个专辑，
我就会自动把它加入到你的菜谱库里，是你亲手选的，肯定更合口味。
```

---

### 消息 5（点击后，用户回复）
```
本地菜谱库这个方式好。收藏专辑也要，
这样刷到好菜直接收藏，很方便。
```

---

### 消息 6（点击后，AI 回复）
```
✅ 收藏专辑已为你创建：「我的菜谱库」

以后在小红书刷到好菜，点击收藏 → 选择「我的菜谱库」专辑，
菜单助手会自动同步这道菜到你的菜谱库，下次规划菜单时就会考虑它。

接下来——菜谱库也支持打分和反馈吧？
这样喜欢的菜可以更快重复出现，不喜欢的减少甚至屏蔽。
```

---

### 消息 7（点击后，用户回复）
```
对，支持每日打分，根据评分动态调整菜品出现频率。
高分的可以更快重复，低分的隔久一点，特别差的直接屏蔽。
```

---

### 消息 8（点击后，AI 回复）
```
最后一个问题：生成菜单后——
是先展示草稿让你确认和调整，还是直接生成最终版？
```

---

### 消息 9（点击后，用户回复）
```
先出草稿，我来确认，可以换菜或者指定某道菜。
```

---

### 消息 10（点击后，AI 回复）
```
完美 ✅ 菜单助手 Skill 定义完成！整理如下：

• 维护本地菜谱库，支持从小红书定期同步新菜
• 创建「我的菜谱库」收藏专辑，收藏即入库
• 每周为周一/二/四/五各规划 3 道菜（1荤+1素+1特色）
• 先出草稿 → 你确认/调整 → 生成购物清单
• 无烤箱，空气炸锅可替代简单烤制
• 支持每日打分，动态调整菜品频率和冷却期

可以先试用一下看看效果 👇
```

---

### 消息 11（点击后，显示试用卡片）

> **UI 元素：试用卡片**
> 样式：圆角卡片，绿色主题
> 内容：
> - 上方：Skill icon（🍽️）+ 名称"菜单助手"+ 简介一行
> - 中间：一行示例提示文字，如"点击试用 → 生成本周菜单"
> - 卡片整体可点击 → 跳转到该 Skill 的对话页（试用模式）
>
> **试用完返回后：**
> - 对话流底部出现"下一步"按钮（绿色）
> - 点击 → 进入 Step 3 Skill 说明书审阅

---

## 备注

- 对话中 AI 头像：绿色点点 logo
- 用户气泡：右对齐，浅灰色背景
- AI 气泡：左对齐，白色背景，左侧绿色竖线装饰
- 每轮点击推进时，AI 回复前可加一个短暂的"正在思考..."动画（1s）
